Deploy blog on github with hexo

Apr 14, 2016 | Hits

Inspired by the blog-talking event of Bittiger, I believe its a good time start writing blogs. Writing blogs is a good way for me to further strengthen the knowledge I gained and share it with others.

Hexo is a great tool to write blog with markdown language, it is powerful but simple and easy to pick up. I feel that it would be great if I could have my blog deployed on github with the domain myusername.github.io. It is helpful if you point out that you have a blog website on resume.

There are already tons of good tutorials on how to build up a hexo blog, so for this tutorial, my foucs will be simple, how to config and deploy the blog on your github page. If you want deploy blog on your own domain, it is super easy to find a tutorial that teches how to do that.

Setting up environment

Hexo is a blog framework powered by Node.js, so make sure that you already have node installed on your machine. Then let’s install hex globally, simply run:

Once you have hexo installed globally, initialize your blog project:

Hexo will generate a folder with your specified blog name, next we will install the tool wich will help us deploy our blog system to github:

Open your github, and create a new repo named yourusername.github.io, github then will automatically launch a website at yourusername.github. It might take up to an hour for github to deploy your page live, so be patient.
Go to config.yml and config your github deploy info. Under deploy session, take my config info as example:

Note when you want to deploy to a project you are working on, simply change branch master to gh-pages.
From now on, whenever you want to write or update your blog post, run hexo generate to synchronise local files and then hexo deploy to deploy updates to your github blog website.

You are almost good to go here, simply run hexo server, and your blog system is now running on localhost:4000.

Usage

To write a new blog post, run

Hexo will generate a file with that name under source _posts folder, go ahead and start write your first blog post!

You can also generate pages such as about in addition to post, hexo generate page pagename would do the trick. Please to refer to your theme configuration for consistency.

Change theme

Of course, hexo default theme is not that appealing, hexo has tons of theme templates.
visit Hexo Theme for theme demos, and Hexo Theme wiki for their actual github repos. Follow the install instruction and change theme name to corresponding name in config.yml file, your hexo server now should run with your new theme.
